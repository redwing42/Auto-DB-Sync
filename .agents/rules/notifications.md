# Email Notification Protocols

## Trigger Rules
- **On New Submission:** Notify all Reviewers/SDEs; [cite_start]BCC Admin. [cite: 153, 154, 155]
- [cite_start]**On Rejection:** Notify the Submitting Operator; include the mandatory rejection reason verbatim. [cite: 161, 162, 164, 179]
- [cite_start]**On Approval:** Notify the Operator with the git branch name and a summary of DB updates. [cite: 165, 166, 168]

## Content Requirements
- [cite_start]All emails must include a direct deep link to the submission in the dashboard. [cite: 181]
- [cite_start]The sender’s full name and role must be visible in the email body. [cite: 178]
